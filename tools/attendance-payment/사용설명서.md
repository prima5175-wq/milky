# 학원 출결·결제 관리 시트 (구글시트 자동화)

기존에 쓰시던 구글시트(출석 날짜 직접 입력 방식)를 그대로 살리면서,
**손으로 적던 회차·금액·결제일을 자동화**한 도구입니다.

---

## 무엇이 좋아지나요?

| 기존(수작업) | 개선 후(자동) |
|---|---|
| K열에 `주2회 90분 분기` 직접 타이핑 | 횟수·시간·납부를 **드롭다운**으로 선택 |
| 금액(J)·총회차·남은회차(O) 직접 계산 | **금액·총회차·다음결제일 자동 계산** |
| 회차 칸 색을 직접 칠함 | **시간별 색 자동** (60분 분홍·90분 초록·120분 노랑) |
| 결제일을 달력 보며 체크 | **다음결제일 자동** + 임박 학생 빨강 강조 |
| 누가 회차 안 쓰는지 눈으로 찾기 | **대시보드에 소진 위험 학생 자동 정리** |

---

## 설치 방법 (5분)

1. 새 구글시트를 하나 만듭니다. (또는 기존 시트 사본)
2. 상단 메뉴 **확장 프로그램 ▸ Apps Script** 클릭.
3. 기본 `Code.gs` 내용을 지우고, 이 폴더의 **`Code.gs` 전체**를 붙여넣고 저장(💾).
4. 구글시트로 돌아와 새로고침하면 상단에 **📚 학원관리** 메뉴가 생깁니다.
5. **📚 학원관리 ▸ ① 시트 초기화/설치** 클릭. (처음에 권한 승인 1회 필요)
   - `출결관리` / `플랜단가` / `대시보드` / `사용안내` 시트가 자동 생성됩니다.

---

## 사용법

### 1) 학생 등록
- `출결관리` 시트에 **이름·학년·학교·연락처·등록일** 입력.
- **횟수**(주1·2·3회/매일반) · **시간**(60·90·120분) · **납부**(월납/분기납) 드롭다운 선택.
- 선택하는 순간 → **금액·총회차·다음결제일**이 채워지고 **회차 칸이 색칠**됩니다.
  - 예) `주1회 90분 월납` → 총회차 **4**, 초록 칸 4개 (1줄)
  - 예) `주2회 90분 분기납` → 총회차 **24**, 초록 칸 8개씩 **3줄**
  - `매일반` → 3줄 가득(날짜 기록용), 남은회차 대신 **출석 횟수**가 카운트됩니다.

### 2) 출석 체크 (두 가지 모두 지원)
- **방법 A** — 회차 칸에 온 날짜를 직접 입력 (예: `3/4`).
- **방법 B** — 칸(여러 개 선택 가능)을 고른 뒤 **📚 학원관리 ▸ ✅ 선택 칸 오늘 출석 체크**.
- 날짜가 든 칸은 **회색(사용됨)** 으로 변합니다 → **남아있는 색칸 = 남은 회차**.
- 취소: **📚 학원관리 ▸ ↩️ 선택 칸 출석 취소** (또는 칸 내용 삭제).

### 3) 결제 / 소진 관리 — `대시보드` 시트
- **결제 임박/지난 학생**: 다음결제일이 7일 이내거나 지난 학생을 자동 정렬.
- **회차 소진 위험**: 남은 기간 동안 평소 페이스로 와도 등록 회차를 다 못 쓰는 학생
  → **연락해서 자주 오게 해야 할 대상**이 자동으로 뜹니다.
- 상단 KPI: 총 등록생 / 이번달 결제예정 합계 / 결제임박 인원 / 소진위험 인원.

### 4) 금액표 — `플랜단가` 시트
- 횟수+시간+납부 조합별 금액표입니다. **실제 학원 금액으로 숫자만 수정**하세요.
- (현재 값은 업로드해주신 시트에서 추정한 예시 금액입니다. 예: 주1회90분분기 48만 등)
- 금액을 바꾼 뒤 **📚 학원관리 ▸ 🔄 전체 다시 계산**을 누르면 일괄 반영됩니다.

### 5) 재등록 / 학생 추가
- 다음 분기·월 재등록 시: 같은 행에서 **등록일/횟수/납부만 바꾸면** 회차 칸이 새로 생성됩니다.
  - ⚠️ 횟수·시간·납부를 바꾸면 회차 칸이 **새로 생성**되어 기존 출석 표시는 초기화됩니다.
    (등록일만 바꾸면 출석은 유지되고 결제일만 갱신됩니다.)
- 학생이 더 필요하면 **📚 학원관리 ▸ ➕ 학생 50명 행 추가**.

---

## 색상 규칙
- **60분 = 분홍** · **90분 = 초록** · **120분 = 노랑**
- **회색 = 출석 완료(사용됨)** · 남은 색칸 = 남은 회차
- **다음결제일** 칸: 7일 이내 주황, 지났으면 빨강 자동 강조

---

## 참고 / 한계
- 한 학생은 **3줄**을 차지합니다(분기납·매일반의 3줄 대응). 플랜 드롭다운은 **맨 윗줄**에서 고르세요.
- 다음결제일은 `등록일 + 1개월(월납)` 또는 `+3개월(분기납·매일반)` 기준입니다.
- 기존 200명 데이터의 **자동 이관**(K열 `주2회 90분 분기` 텍스트 → 드롭다운/색칸 변환)도 가능합니다.
  데이터가 복잡(한 칸에 여러 플랜·자유 텍스트 금액 등)해 자동 변환 시 검수가 필요하니,
  원하시면 별도로 진행해 드리겠습니다.
