# PRD: BookSpec — 학부모 연동 독서 보상 결제 시스템

**버전**: v0.1
**작성일**: 2026년 4월
**작성자**: 포레스트리 (밀키)
**상태**: Draft

## 1. 제품 개요

### 1.1 Value Positioning

> **BookSpec은 독서논술학원에 자녀를 보내는 초·중등 학부모가 "돈은 내는데 아이가 뭘 하는지 모르는" 상황에서, 종이 독서록과 카카오톡 단톡방에 의존하던 기존 방식보다 — 학부모가 직접 보상을 설계하고, AI가 독서 기록을 세특 포트폴리오로 연결해주는 방식으로 — 아이의 독서 동기와 입시 성과를 동시에 관리하는 문제를 더 잘 해결한다.**

### 1.2 핵심 전제

- 결제자(학부모)와 사용자(자녀)가 분리된 구조
- 학원·공부방 원장을 B2B 채널로 활용하는 B2B2C 모델
- 텍스트 전용 AI 인터랙션으로 MVP 구현
- 우주·행성 테마 세계관 ("질문이 스펙이다")

---

## 2. 배경 및 문제 정의

### 2.1 시장 맥락

- 한국 독서논술학원 수강 초등 학부모: 30~50만명 (강남·서초·대치 핵심)
- 전국 독서논술 학원·공부방: 3,000~5,000개소
- 전자출판 플랫폼 시장 전년 대비 12% 성장 (2024)
- 2024년 성인 독서율 87.8%로 반등, '텍스트힙' 트렌드 부상

### 2.2 핵심 공백 3가지

**공백 1 (최우선)**: 학부모가 결제하고 보상을 직접 제어하는 구조가 없음

**공백 2**: AI 독서 요약 + 세특 연결 분석 서비스가 없음

**공백 3**: 원장이 학부모에게 자동 리포팅하는 B2B 운영 도구가 없음

## 3. 타겟 사용자

### 3.1 주요 타겟 (결제자)

**페르소나 A — 박수진 (P1, 최우선)**
- 42세, 서초구 전업주부, 초등 4학년 딸
- 핵심 Need: 실시간 모니터링 + 내 손으로 보상 지급

**페르소나 B — 이미경 (P2)**
- 38세, 평택 간호사 3교대, 초등 3학년 아들
- 핵심 Need: 바쁜 일상 중 비동기 관리 + 실제 쓸 수 있는 보상

**페르소나 C — 정혜영 (P3)**
- 46세, 강남 카페 사장, 중학교 2학년 아들
- 핵심 Need: AI가 독서 내용을 세특 포인트로 자동 분석

### 3.2 B2B 타겟 (채널)

**페르소나 D — 강은주 (원장)**
- 49세, 노원구 독서논술 공부방, 수강생 15명
- 핵심 Need: 수강생 대시보드 + 학부모 자동 리포팅
